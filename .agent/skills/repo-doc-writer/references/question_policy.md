# Question Policy for repo-doc-writer

The skill may ask clarification questions, but only when those answers materially affect documentation correctness.

## Ask questions when
- architecture boundaries are unclear
- a workflow may differ between local, notebook, or deployment environments
- a public interface or artifact contract may be wrong without clarification
- ownership between modules cannot be inferred
- the user wants docs for a business process that is not visible in code

## Do not ask questions when
- the answer is inferable from filenames, code, comments, or CLI args
- the uncertainty is cosmetic
- a reasonable assumption can be stated explicitly
- the missing detail does not affect file structure, interface docs, or operational guidance

## Limits
- ask at most 5 questions at a time
- group related questions
- if no answer is strictly required, proceed immediately with labeled assumptions
