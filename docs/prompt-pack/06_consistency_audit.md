# Prompt 06 — Documentation Consistency Audit

Use `repo-doc-writer` to review the documentation set for consistency and completeness.

Check for:
- conflicting statements across files
- duplicated information that should have one source of truth
- unsupported claims not grounded in the repo
- stale commands or artifact names
- missing cross-links between architecture, module docs, and contract docs
- hidden assumptions still present in code but absent from docs

Required output:

## Consistency check
### Critical issues
- ...

### Important issues
- ...

### Nice-to-have improvements
- ...

## Suggested patches
- `path/to/file.md`: what to change
- ...

If the docs are already reasonably consistent, say so.
