# Prompt 03 — Write a Module README

Use `repo-doc-writer` to write a README for this module: `[MODULE_PATH]`

The README must include:
- what this module is responsible for
- what it is not responsible for
- important files and subfolders
- main entry points
- input and output artifacts
- commands or invocation patterns if inferable
- what to test after making changes here
- visible technical debt, TODOs, or ambiguity from the code

Rules:
- inspect the real files under `[MODULE_PATH]`
- avoid generic folder summaries
- prefer contracts and operational guidance over prose
- ask me questions only if the answer materially changes the README
- if commands are inferred, label them as inferred

Before writing, show:
- outline
- assumptions
- blocking questions if any

Then draft the README.
