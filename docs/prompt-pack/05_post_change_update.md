# Prompt 05 — Update Docs After a Code Change

Use `repo-doc-writer` to update documentation after these code changes:
`[PASTE SUMMARY OR DIFF CONTEXT HERE]`

Tasks:
1. Identify which docs are now stale.
2. Update only the docs that need to change.
3. If the change affects artifact names, schemas, CLI signatures, or runtime workflow, call that out explicitly.
4. Add change-impact notes where helpful.
5. Keep the canonical source of truth minimal; do not duplicate unnecessarily.

Question policy:
- ask at most 5 questions
- only ask if the answer materially changes documentation correctness
- otherwise proceed with assumptions

Before editing, show:
- docs that need updates
- why they need updates
- assumptions
- blocking questions if any

Then provide the updated markdown.
